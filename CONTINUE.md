# slsa-verifier — Continue Context

## Project Overview

slsa-verifier is a Go-based CLI tool for verifying [SLSA provenance](https://slsa.dev/provenance/) generated by CI/CD builders. It cryptographically verifies signatures on provenance attestations to ensure they were created by the expected builder, and validates that metadata (builder ID, source repository, ref/branch/tag) matches expected values.

**Module:** `github.com/slsa-framework/slsa-verifier/v2`  
**Language:** Go 1.23.2+  
**License:** Apache 2.0

- **Sigstore stack:** rekor, cosign, fulcio, sigstore-go, protobuf-specs
- **In-Toto:** attestation (v1.1.0), in-toto-golang (v0.9.0)
- **Container:** google/go-containerregistry, cosign/v2
- **CLI:** spf13/cobra, spf13/pflag
- **Build system:** slsa-github-generator v1.10.0

## CLI Commands

| Command | Description |
|---------|-------------|
| `verify-artifact` | Verify SLSA provenance on binary artifact blobs |
| `verify-image` | Verify SLSA provenance on container images |
| `verify-npm-package` | [Experimental] Verify npm package attestations |
| `verify-github-attestation` | Verify GitHub `attest-build-provenance` attestations |
| `verify-vsa` | Verify Verification Summary Attestations (VSAs) |

### Common Flags

- `--source-uri` — Expected source repository (e.g., `github.com/org/repo`)
- `--source-tag` — Expected tag the binary was compiled from
- `--source-branch` — Expected branch
- `--source-versioned-tag` — Semantic version matching for tags
- `--builder-id` — Unique builder ID who created the provenance
- `--provenance-path` — Path to provenance file
- `--print-provenance` — Print verified provenance to stdout

## Common Tasks

### Adding a new verifier

1. Define CLI command in `cli/slsa-verifier/verify.go` using cobra
2. Implement verification logic in `verifiers/verifier.go`
3. Register the verifier in `register/register.go`
4. Add regression tests in `cli/slsa-verifier/main_regression_test.go`
5. Add test data under `cli/slsa-verifier/testdata/`

### Working with provenance

- Provenance follows the [SLSA provenance format](https://slsa.dev/provenance/)
- Signatures use Sigstore (Rekor + Fulcio) for transparency
- Container images must use immutable digests to prevent TOCTOU attacks
- Multiple artifacts can share a single provenance file (all must be in `subject`)

### Environment Variables

- `SLSA_VERIFIER_EXPERIMENTAL=1` — Enable experimental features (npm packages)
- `GH_TOKEN` — Required for GitHub Container Registry (ghcr.io) access
- `COSIGN_REPOSITORY` — Override for provenance repository (container images)
